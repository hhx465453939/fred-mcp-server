# Full Endpoint Tools

In addition to the 3 “core” tools (`fred_browse`, `fred_search`, `fred_get_series`), this server also exposes **one-to-one MCP tools** that map directly to the official St. Louis Fed Web Services endpoints:

- **FRED API v1**: `https://api.stlouisfed.org/fred/...` ([official docs](https://fred.stlouisfed.org/docs/api/fred/))
- **GeoFRED (Maps API)**: `https://api.stlouisfed.org/geofred/...` ([official docs](https://fred.stlouisfed.org/docs/api/geofred/))
- **FRED API v2**: `https://api.stlouisfed.org/fred/v2/...` ([official docs](https://fred.stlouisfed.org/docs/api/fred/v2/index.html))

## How parameters work

Most tools follow this pattern:

- A required identifier (e.g. `series_id`, `release_id`, `source_id`, `category_id`)
- An optional `params` object for passing **any additional official query parameters** through to the underlying endpoint

This makes the tools future-proof and keeps the MCP schemas stable while still supporting the full official parameter surface.
